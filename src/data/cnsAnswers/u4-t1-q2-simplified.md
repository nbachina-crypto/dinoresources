
SHA converts any input into a **fixed-size hash value**.
---
## Steps (Simplified)
1. Pad the message
2. Divide into blocks
3. Process each block using:
   * Bitwise operations
   * Shifts and rotations
4. Update intermediate values
5. Combine results to produce final hash
---
## Key Idea
> Even a **small change in input** produces a completely different hash, making SHA highly secure and reliable for cryptographic applications.
