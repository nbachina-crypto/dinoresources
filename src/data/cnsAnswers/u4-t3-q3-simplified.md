
An **X.509 certificate** is a digital document that links a **public key** to an identity.
---
## How It Works
* A **CA issues and signs** the certificate
* A client verifies it using the **CA’s public key**
* If valid, the public key is trusted
---
## Example
* A website sends its certificate in HTTPS
* The browser verifies it
* Secure communication begins
---
> **Key Idea:** X.509 certificates enable trusted public key distribution by binding identities to keys through **CA-signed certificates**.
